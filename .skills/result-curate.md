# result-curate

## 1. 目标

将本次 run 的过程结果整理为长期可复用资产。

## 2. 输入

- 设计文档
- 实现计划
- 切片结果
- 验证报告
- 偏差记录

## 3. 执行动作

- 更新追踪矩阵
- 归档验证报告
- 汇总偏差与未决项
- 生成最终交付索引

## 4. 输出

- final-index.md
- traceability-matrix.md
- deviations-summary.md
- next-iteration-context.md

## 5. 关键要求

沉淀结果应支持下一轮 AI 或人工继续增量推进，而非重新理解全量上下文。
