# design-normalize

## 1. 目标

将设计资料统一收敛为结构化 Markdown 设计文档。

## 2. 输入

- 原始设计 Markdown
- PDF / 补充说明
- intake 结果

## 3. 执行动作

- 合并补充信息
- 标准化章节结构
- 补齐术语、约束、接口和验收描述
- 标记待确认项

## 4. 输出

- normalized-design.md
- normalization-report.md
- unresolved-items.md

## 5. 门禁

输出文档必须满足模板要求，才允许进入 plan 阶段。
