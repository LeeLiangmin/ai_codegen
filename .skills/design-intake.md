# design-intake

## 1. 目标

验证输入设计文档是否满足 AI 可执行要求，并输出 intake 结果。

## 2. 输入

- 设计文档 Markdown
- 可选参考资料（如 PDF）
- 项目上下文说明

## 3. 检查项

- 是否包含目标、范围、模块设计、接口设计、数据设计、验收标准
- 是否存在明显冲突或缺失
- 是否存在无法落地的模糊表述
- 是否缺少 AI 实施边界说明

## 4. 输出

- intake-report.md
- 缺失项清单
- 风险项清单
- 是否允许进入 normalize 阶段的结论

## 5. 失败处理

若存在关键章节缺失，停止进入后续阶段。
