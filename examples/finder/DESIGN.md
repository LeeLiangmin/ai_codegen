## 2. 背景与目标

### 2.1 业务背景

需要在仓库 `examples/finder` 中沉淀一个**小而完整**的查找能力示例，便于演示设计驱动流程与后续实现。典型场景包括：在文件内容中搜索关键字或模式，以及按名称/类型/深度等条件定位文件与目录。

### 2.2 目标

1. **文本查找**：在指定根目录下扫描文本类文件，按字面或正则匹配内容，输出命中位置（文件路径、行号、行内容片段）。
2. **文件/目录查找**：在指定根目录下按名称模式（支持通配或简单 glob）、类型（文件/目录/二者）、可选最大深度等条件枚举路径。

### 2.3 非目标

- 全文索引、增量索引或后台守护进程式「监视」。
- 压缩包内搜索、Office/PDF 等非纯文本二进制解析。
- 跨机器分布式搜索或网络文件系统专用优化。
- GUI；本设计以 **CLI（命令行）** 为主，程序库 API 为可选扩展。

---

## 3. 范围

### 3.1 In Scope

- 单进程、本地文件系统上的递归遍历（可配置最大深度、是否跟随符号链接，默认**不跟随**以降低环与意外范围风险）。
- 文本查找：UTF-8 为主；对无法解码为文本的文件可跳过或记录为「跳过」类结果（实现时二选一并文档化）。
- 文件/目录查找：名称过滤、文件/目录筛选、深度限制、可选「仅列出文件数/目录数」统计模式（可选，若工期紧可放入后续迭代）。
- 统一的退出码约定：成功 / 无匹配 / 用户错误 / 运行时错误（具体数值在实现阶段确定）。
- `README.md` 中与上述能力一致的使用说明（若本仓库 examples 惯例需要）。

### 3.2 Out of Scope

- 编辑器集成、LSP、IDE 插件。
- 权限提升、绕过系统访问控制。
- 复杂查询语言（如 SQL）；仅保留 CLI 标志位与简单模式字符串。

## 4. 术语与约束

### 4.1 术语定义

| 术语 | 定义 |
| --- | --- |
| 根目录 | 用户指定的搜索起点路径，所有递归均限制在其子树内（除非另有说明）。 |
| 文本查找 | 按行（或按实现文档说明的粒度）在文件内容中匹配模式。 |
| 命中 | 一次成功的模式匹配，带文件路径与位置信息。 |
| 名称模式 | 用于匹配路径最后一段（文件名或目录名）的字符串，可为字面、通配或 glob（实现阶段固定一种并写清）。 |
| 深度 | 根目录为 0，每向下一层 +1。 |

### 4.2 全局约束

- **技术栈**：由实现阶段选定（推荐 Python 3.10+ 或 Node.js LTS），需与仓库 `examples` 目录惯例一致；若仓库无惯例，优先 **Python**，依赖保持最少。
- **安全**：禁止将用户输入直接拼进 shell 执行；路径应规范化并限制在根目录解析结果之下（防止 `..` 逃逸）。
- **性能**：在数千小文件量级下可接受秒级完成；不承诺万级以上目录的实时性保证，但应避免明显的 O(n²) 重复读盘。
- **合规**：不收集、不上传路径或文件内容；日志若含路径需注意敏感目录提示（文档层面）。

## 5. 业务流程

### 5.1 主流程 — 文本查找

1. 解析 CLI：根目录、模式（regex 或 literal 标志）、包含/排除 glob（可选）、最大深度（可选）。
2. 校验根目录存在且为目录。
3. 遍历文件树（按深度策略与过滤规则）。
4. 对候选文件尝试按文本打开并逐行匹配。
5. 将命中按统一格式输出到 stdout。
6. 汇总退出码（有命中 / 无命中）。

### 5.2 主流程 — 文件/目录查找

1. 解析 CLI：根目录、名称模式、类型（file/dir/both）、最大深度（可选）。
2. 校验根目录。
3. 遍历并过滤名称与类型。
4. 输出路径列表（每行一条或实现选定格式）。
5. 设置退出码。

### 5.3 异常流程

| 场景 | 处理 |
| --- | --- |
| 根路径不存在或非目录 | 错误信息到 stderr，非零退出。 |
| 模式非法（正则编译失败等） | 错误信息到 stderr，非零退出。 |
| 单文件不可读 | 跳过该文件，可选 stderr 警告；整体仍可进行。 |
| 遍历遇环（符号链接） | 默认不跟随 symlink；若未来支持跟随，需记录已访问 inode/真实路径防环。 |
| 用户中断（Ctrl+C） | 优雅退出，非零码。 |

## 6. 模块设计

### 6.1 模块：CLI 入口

- **职责**：参数解析、帮助信息、调用各子命令并处理退出码。
- **输入**：`argv`、环境变量（可选，如 `NO_COLOR`）。
- **输出**：进程退出码；stdout/stderr 文本。
- **依赖**：文本查找模块、路径查找模块、公共配置类型。
- **边界**：不做业务遍历逻辑，仅编排。
- **错误处理**：参数错误立即失败；运行时错误带简短原因。
- **安全要求**：校验路径在根之下。

### 6.2 模块：文件树遍历（Walker）

- **职责**：自顶向下枚举路径，应用深度与类型过滤，可选 symlink 策略。
- **输入**：根路径、选项（深度、类型、是否 follow symlink）。
- **输出**：路径迭代器或回调流。
- **依赖**：标准库文件系统 API。
- **边界**：不负责内容匹配。
- **错误处理**：权限错误可跳过并警告。
- **安全要求**：路径 join 后必须仍在根子树内（规范化后比较）。

### 6.3 模块：文本查找（ContentSearch）

- **职责**：对给定文件路径按行读取并应用模式；产生结构化命中记录。
- **输入**：文件路径、模式、是否 regex、可选上下文行数（后续迭代）。
- **输出**：命中列表（路径、行号、行文本）。
- **依赖**：Walker 或上层传入的文件列表。
- **边界**：不解析二进制格式。
- **错误处理**：编码错误按全局策略跳过或失败。
- **安全要求**：限制单行输出长度，避免内存爆炸（可选上限）。

### 6.4 模块：路径查找（PathSearch）

- **职责**：在 Walker 输出上按名称模式过滤。
- **输入**：根、名称模式、类型过滤、深度。
- **输出**：路径列表。
- **依赖**：Walker。
- **边界**：不读取文件内容。
- **错误处理**：同 Walker。
- **安全要求**：同 Walker。

### 6.5 模块：公共类型与格式化（Output）

- **职责**：命中/路径的统一打印格式（如 `path:line:content` 或 ripgrep 风格子集）。
- **输入**：结构化记录。
- **输出**：字符串行。
- **依赖**：无或仅标准库。
- **边界**：不含业务逻辑。
- **错误处理**：—
- **安全要求**：对控制字符可做转义或剥离（可选）。

## 7. 数据设计

### 7.1 核心实体（逻辑模型）

| 实体 | 字段 | 类型（逻辑） | 必填 | 约束 | 备注 |
| --- | --- | --- | --- | --- | --- |
| SearchRoot | path | string | 是 | 必须为绝对或经解析后的规范路径 | 搜索根 |
| SearchOptions | maxDepth | int \| null | 否 | >= 0 | null 表示不限制 |
| SearchOptions | followSymlinks | bool | 否 | 默认 false | |
| TextHit | filePath | string | 是 | | |
| TextHit | lineNumber | int | 是 | >= 1 | 1-based |
| TextHit | lineText | string | 是 | | 可截断 |
| PathHit | path | string | 是 | | 文件或目录 |
| PathHit | kind | enum | 是 | file \| directory | |

### 7.2 数据关系

- 一次 CLI 调用对应一个 `SearchRoot` 与一份 `SearchOptions`。
- 多个 `TextHit` / `PathHit` 可属于同一次调用，彼此独立，无需持久化。

### 7.3 数据迁移需求

无（无数据库）。

## 8. 接口设计

本示例以 **CLI** 为主接口；若实现为库，下列子命令可映射为函数。

### 8.1 命令：`finder text`（或 `finder grep`）

| 项 | 说明 |
| --- | --- |
| 目的 | 在目录树内按内容查找 |
| 形式 | CLI 子命令 |
| 参数 | `<root> <pattern>`；`--regex` / `--fixed-strings`（二选一，默认其一并在 README 写明）；`--max-depth N`；`--glob INCLUDE`；`--glob-exclude EXCLUDE`（可选） |
| 输出 | stdout：每命中一行，建议 `path:line:content` |
| 错误 | stderr 人类可读；非法参数非零退出 |
| 幂等 | 是（只读） |
| 权限 | 读取用户可读文件 |

### 8.2 命令：`finder path`（或 `finder find`）

| 项 | 说明 |
| --- | --- |
| 目的 | 按名称与类型查找路径 |
| 形式 | CLI 子命令 |
| 参数 | `<root>`；`--name PATTERN`；`--type f|d|a`（file/directory/all）；`--max-depth N` |
| 输出 | stdout：路径每行一条 |
| 错误 | 同 8.1 |
| 幂等 | 是 |
| 权限 | 列目录与 stat |

### 8.3 命令：`finder --help` / `finder <subcommand> --help`

- 列出子命令与关键选项。

## 9. 状态与规则

### 9.1 状态机

无长期状态；单次调用为**无状态**扫描。

### 9.2 核心业务规则

1. 所有相对路径均相对于进程当前工作目录解析后再与根目录规则结合（实现时明确优先级）。
2. 默认**不跟随**符号链接。
3. 文本查找默认跳过明显二进制（如含 NUL 字节的前 8KB 检测）若实现支持该启发式；否则仅按扩展名或 MIME 简单过滤（可选，写入 README）。
4. 匹配计数与退出码：建议「有命中 → 0，无命中 → 1」与 grep 习惯一致，或统一文档化另一种约定。

## 10. 验收标准

### 10.1 功能验收

- [ ] `finder text`：在已知fixture目录中能找到含指定字符串/正则的文件与行号。
- [ ] `finder path`：能列出匹配名称的文件、目录或二者。
- [ ] `--max-depth` 生效（可通过 fixture 目录结构验证）。
- [ ] 非法根路径、非法正则时失败且退出码非零。
- [ ] `--help` 可读且与实现一致。

### 10.2 质量验收

- 提供最小 fixture 目录或生成脚本，便于自动化测试。
- 单元测试覆盖：路径规范化、名称匹配、至少一种内容匹配。
- Windows 与 POSIX 路径分隔符行为在 README 中说明（若仅先支持一方亦需写明）。

### 10.3 设计一致性要求

- 不得在未更新设计的情况下引入第三大类主功能（如网络搜索）。
- 安全规则（路径逃逸、symlink）不得弱化；若变更须在本文档修订并注明版本。

## 11. 风险与待确认项

### 11.1 风险

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| 大文件单行过长 | 内存占用 | 单行截断或流式处理 |
| 编码非 UTF-8 | 漏检或乱码 | 明确策略（跳过/替换字符/检测编码） |
| Windows 路径大小写与保留名 | 行为与 Unix 不一致 | README 与测试用例覆盖 |

### 11.2 待确认项

1. 最终实现语言与包管理器（pip / npm / 其他）。
2. 文本查找默认 **regex** 还是 **固定字符串**。
3. 名称匹配采用 **glob**、**fnmatch** 还是 **正则**（三选一作为 v1）。
4. 无匹配时退出码为 0 还是 1（与类 grep 工具对齐与否）。

## 12. AI 实施附录

### 12.1 实现切片建议

1. **Slice A**：公共路径工具（规范化、根下校验、Walker + 深度与 symlink 策略）。
2. **Slice B**：`finder path` 子命令与测试 fixture。
3. **Slice C**：`finder text` 子命令与编码/二进制启发式策略。
4. **Slice D**：README、退出码统一、CI 可运行的测试命令（若仓库已有 CI）。

### 12.2 推荐测试点

- 空目录、仅文件、仅子目录、混合深度。
-  symlink 指向树外（应不跟随或拒绝）。
-  含空格与 Unicode 的文件名。
-  正则特殊字符在 fixed-string 模式下按字面匹配。

### 12.3 禁止事项

- 不得在示例代码中硬编码用户机器上的绝对路径作为默认搜索路径。
- 不得静默吞掉所有 IO 错误；至少统计或警告。
